# 前端优化记录

## 已完成的优化

### 1. 节点拖拽功能修复

**问题描述**: 节点无法自由移动位置

**原因分析**:
- 画布拖拽和节点拖拽使用同一个 `isDragging` 状态
- 条件判断顺序导致节点拖拽逻辑永远不会执行

**解决方案**:
- 在 `canvas.ts` 中新增 `isDraggingNode` 状态
- 新增 `startDraggingNode()` 方法专门处理节点拖拽
- 修改 `handleMouseMove` 中的条件判断顺序，优先处理节点拖拽

**涉及文件**:
- `frontend/src/stores/canvas.ts`
- `frontend/src/components/WorkflowCanvas.vue`

---

### 2. 节点删除功能

**问题描述**: 无法删除节点

**原因分析**:
- Store 有 `deleteNode` 方法，但 UI 没有调用
- 缺少键盘事件监听

**解决方案**:
- 添加键盘事件监听，支持 `Delete` / `Backspace` 键删除
- 在 PropertyPanel 中添加删除按钮

**涉及文件**:
- `frontend/src/components/WorkflowCanvas.vue`
- `frontend/src/components/PropertyPanel.vue`

---

### 3. 连线选择与删除

**问题描述**: 连线无法点击选择

**原因分析**:
- SVG 层设置了 `pointer-events: none`，导致无法响应点击事件

**解决方案**:
- 移除 `.connections-layer` 的 `pointer-events: none` 样式
- 添加键盘删除连线功能

**涉及文件**:
- `frontend/src/components/WorkflowCanvas.vue`

---

### 4. 属性面板消失问题

**问题描述**: 点击节点后，右侧属性配置面板很快消失

**原因分析**:
- 点击节点时，`click` 事件冒泡到画布
- 画布的 `handleCanvasClick` 清除了选择状态

**解决方案**:
- 在 WorkflowNode 根元素添加 `@click.stop` 阻止事件冒泡

**涉及文件**:
- `frontend/src/components/WorkflowNode.vue`

---

### 5. 画布平移功能

**问题描述**: 只能缩放画布，无法拖动整个画布

**原因分析**:
- `handleMouseDown` 中的判断条件 `e.target === canvasRef.value` 过于严格
- 画布内部有子元素覆盖，点击时 target 不是画布本身

**解决方案**:
- 改为检查点击目标是否在节点、连线、端口或缩放控件内
- 如果不是这些元素，则允许画布拖拽

**涉及文件**:
- `frontend/src/components/WorkflowCanvas.vue`

---

### 6. 重置视图功能

**新增功能**: 一键将画布居中到节点区域

**实现方案**:
- 计算所有节点的边界框
- 计算中心点并设置偏移量
- 重置缩放为 100%

**涉及文件**:
- `frontend/src/components/WorkflowCanvas.vue`
- `frontend/src/stores/canvas.ts`
